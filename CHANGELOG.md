# Changelog

All notable changes to `@julianoczkowski/agentwiki` are documented here.

## 1.0.0 — 2026-07-16

### Monorepo support 🎉

`init` now asks **"Which app should the wiki document?"** when it detects a monorepo, and the whole pipeline honors the choice.

- **Interactive app picker** — a fresh `agentwiki init` in a monorepo opens an arrow-key select listing the detected applications (plus "The whole repository"). Single-project repos never see the question.
- **Apps vs shared packages** — only *applications* are listed; shared packages/libraries (`packages/ui`, `libs/*`, tsconfig presets, …) sit behind a "Something else…" option so the list stays clean. Classification uses NX `project.json` (`projectType`), `nx.json` `workspaceLayout`, and path heuristics, in that order.
- **Broad layout detection** — npm/pnpm workspace globs, NX (including custom layouts like `clients/apps` and `project.json`-only apps), and a manifest sweep up to three levels deep that also covers Go, Rust, and Python monorepos with no JS workspace config.
- **Scoped facts everywhere** — with a scope saved, scanning, symbol extraction, the module graph, and git facts (hot files, recent commits, 90-day counts) cover only that app. File paths in the wiki are app-relative; the quickstart carries a "Monorepo scope" fact and the enrich prompt tells your agent which app it is documenting.
- **`init --scope <dir>`** — set or change the scope non-interactively (`--scope .` returns to whole-repo mode). The choice is stored in `agentwiki/.agentwiki.json`, so `update` — including Cursor/Claude stop-hooks and CI — respects it silently with no prompts.
- **Visible everywhere** — `status` and `doctor` report the active scope.

All existing guarantees hold under a scope: repeated `update` stays a byte-identical no-op, commits to *other* apps never stale your prose, and prose written by your agent is never lost.

### Why 1.0.0

The `.agentwiki.json` metadata gained a `scope` field and `ScanResult`/`GitFacts`/`buildEnrichPrompt` signatures changed. Wikis generated by 0.x keep working unchanged — whole-repo remains the default.
